# Отчет по DAG для такси-сервиса

## Кайсина Полина Вариант: январь 21-31
## Тема: Заказ такси
## Сущности: Клиенты, водители, автомобили, поездки, тарифы, оплата, оценка поездки

### Краткое описание
Данный проект содержит DAG `taxi_order_dag.py`, который моделирует поток сервисного заказа такси: генерация сообщений, проверка их на валидность и формирование отчета по результатам.

### Ход работы
- Вначале настроила WSL2 и установила Ubuntu 22.04, чтобы работать из Linux-окружения на Windows.
- Установила Python в Ubuntu и подготовила виртуальное окружение.
- Открыла проект в VS Code и убедилась, что `taxi_order_dag.py` лежит в корне.
- Написала DAG с задачами:
  - генерация тестовых сообщений в JSON;
  - проверка сообщений на наличие ключей `client_id`, `driver_name`, `car`, `price`, `rating`;
  - подсчет валидных и невалидных поездок;
  - вывод итогового резюме.
- Запустила Airflow Web UI и активировала DAG.
- Запустила DAG вручную из Airflow и проверила файлы `taxi_messages.json`, `taxi_validated.json` и `taxi_validation_summary.json`.

### Результат выполнения
- DAG успешно выполнился, все задачи прошли.
- Результат в Airflow показал последовательное выполнение: `generate_taxi_messages` → `consume_and_validate_messages` → `archive_validation_results` → `print_validation_summary`.
- В отчете появились статистика по валидным/невалидным поездкам.

### Отдельные файлы
- `taxi_order_dag.py` — основной Airflow DAG для проекта такси.
- С помощью `generation.py` в DAG генерируются случайные тестовые поездки.
